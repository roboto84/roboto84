Geometrize Your Images

16 Nov 2022, updated on 28 Jul 2023 ‚Ä¢ 7 MIN READ

Introduction #

As a data-conscious Web Developer, I avoid using images in my applications when I can. If I do use them, I try to use them with impact while handling them carefully in data storage and through the wire. Why so careful? Images are big and don't scale as well as text data. Try managing and storing 10,000 words, that's pretty easy... then consider trying to manage and store 10,000 images. I bet managing the images scenario will make you sweat a bit more üòÖ. We should think twice about how we use such an important form of storytelling on the web to get our message across quickly and with deep impact. There are multiple options to load and present images on the web, e.g, different image formats, image compression, lazy loading, above the fold strategies. One of my favorite options is leveraging vector images. Specifically, vector images based on the geometrization of raster images, due to their smaller file size, packaging options, and cyber-punkish art form (vibes üòè).

Vector Images #

Vector images are great. They can be fairly small in storage size and infinitely resizable. In a basic explanation, vector images represent data about the image as formulas that describe how the image should be drawn using vectors. How do you draw an image with only vectors? You can think of a vector as a line with a certain length for our purposes here. These lines are used to create geometric shapes that are usually filled in with colors. These shapes (they could be hundreds or thousands of them) make up the image we want the file to portray. High quality images on the web are usually not in vector formats (jpg, gif, webp, avif, png) due to how much information some images need to convey (more on that later). These high quality images are stored as what is known as raster formats. Raster images represent data about the image not as vector formulas, but as a 2D grid of pixels.


Vector image vs rastor image representation

Fonts are a great example of vector vs raster formats. Raster formats don't resize very well due to their image data being heavily based on pixel definition data. Imagine what a nightmare it would be to resize fonts if they were stored in raster formats? It would make the text illegible at some point.


Vector font vs rastor font representation

Clearly the vector format is ideal for some use cases, but that isn't the case for all images. Take for example a high fidelity image of Copenhagen's Nyhavn Canal.


Raster image of Copenhagen's Nyhavn Canal

This image has a lot of information. At some point, if an image has too many details, describing it in terms of vectors becomes impractical as you would need to define vector shapes that contain less and less pixels when compared to its raster format equivalent. That is something worth considering when comparing both formats. Vector formats are more efficient for images in which the average vector shape definition can define the placement of multiple pixels of a raster format equivalent as long as the average vector shape definition takes less space than the pixel definition data. Geometrized Images #

Now that we understand how images are stored in vector and raster formats, let's talk about geometrizing raster images to get vector approximates. In order to produce these vector images, we can use some great tools that have already been created by talented developers for this very purpose. These tools will transform our raster format images into Scalable Vector Graphics (SVG). Specifically, you can choose to use one of the following:

  1. Geometrize
  2. primitive.js
  3. Primitive (for macOS)

I'm a Linux person. Fedora being my daily driver, I used Geometrize to convert my images into geometric gold. If you are feeling swirly, you can also use it's scripting interface which leverages the C++ based scripting language, chaiscript. I'll start off by geometrizing this post's thumbnail:

As you can see, there are some interesting results. How well the new vector image comes out is a bit subjective, but we can discuss a couple of factors. We can say that not all images work for this kind of transformation. Nothing is a silver bullet, but the options can be powerful. Usually images with naturally geometric features, natural lines, and a good amount of contrast do better; they allow the algorithm to make better decisions about how to generate shapes more efficiently leading to a powerful geometric based image at a low storage cost.

Conclusion #

It is easy to enjoy these tools for their artistic potential and smaller file size. It is able to reproduce popular art styles for your custom images while also giving you a lot of file size options by controlling how many shapes a generated image uses and by using SVG compression tools such as svgo. These vector approximate images can then be used for a variety of use cases, such as:

  1. Placeholders for large images while they are dynamically loading.
  2. Artful images for your main stage Hero section.
  3. Thumbnails for larger images.
  4. Infinitely re-sizable background images.

As with any tool, it won't be the best option for all images or all situations. It is an interesting option though that leads to even more interesting options as pointed out on Primitive's website...

"The results are somewhat random, so you won't get the same output twice, even with the same inputs. One interesting use of this is to generate multiple images with the same input and then make an animated GIF of them to bring a static image to life."

So there is a lot of room for creativity. For a more in depth reading on how SVG images can be used in support of high fidelity images, consider the following link:
