SSH Server Banner

17 Jul, 2022


Introduction

Lets talk about your SSH Server Banner, and the fact that you don't have one... just yet. No worries, that was me not too long ago. It is easy to configure, but the first question to ask before you do anything is an existential consideration of the matter ...

Well, there are a few reasons to have one:

  • A security warning to let anyone attempting to log into the server you administer know the consequences of their actions.
  • To establish the terms and conditions upon which the server can be used.
  • To present the user with any server information that doesn't require a password, and should be public knowledge (i.e. branding).
  • Dare I say, server ricing 😏 (Don't know what ricing is? Check out the term here: The Basics of Ricing Linux).

If you have concluded to proceed, then let us begin.

Procedure

Edit 'sshd_config'

  • Prepare to edit the server's SSH configuration file sshd_config using sudo vi /etc/ssh/sshd_config.

  • Change the line #Banner none to Banner /etc/issue and save the file. This will activate a banner on an SSH login attempt. The path of the file given can be any of your choosing. In this example the standard path of /etc/issue is used.

  • This concludes all the configuration needed on the SSH service.

Edit Banner File

Edit the banner file with sudo vi /etc/issue and inject the banner that should be presented on a login attempt. The banner file could be a great place to use some ASCII art. I have the following example banner on one of my DEV servers:

For some ASCII text art, check out ASCII art generator | TextKool. I'm a sucker for the ANSI Shadow look 😋.

Restart SSHD

  • ⚠️ WARNING: In order for the changes to take effect, the SSH service will need to be restarted. In modern systems this will not disconnect your connection, but on older systems it might. Assure you did not make any mistakes on your SSH config file or the SSH service may fail to restart, and you may struggle to get back into your server. Validate your sshd_config file with sudo sshd -t. If no errors appear, you should be good to go. Check this link out for more information on checking your SSH configuration file: How to check SSH server's configuration validity

  • Restart the SSH service with sudo service sshd restart

  • Check that the SSH service was restarted, loaded, and active with sudo service sshd status

  • At this point, everything should be good to go. Logout and attempt a login to see your new banner.

Conclusion

With this procedure we have discussed the considerations for a SSH server banner, how to activate it and implement it. Here are some other useful links on the matter:

  • How to Set a Custom SSH Warning Banner and MOTD in Linux

  • HowTo: Set a Warning Message (Banner) in SSH - ShellHacks

Enjoy your cool new banner... 😏 if it is subjectively cool.
