Currently, I enjoy working with network applications, which have an impact on my personal life. These network apps can run on the internet or on a local LAN. Each setup has its pros and cons, but the apps can be configured for either. Currently, I host demos of the apps on the web, but I run the ones I use heavily on my local LAN Fedora server.

roboto

A web platform front-end for a variety of tools. This includes Air, Lexicon, and wh00t. It relies on a REST API and WebSockets for data transfers. It is a minimalist UI with mobile and theming support meant to focus on the data that you care about.

roboto_ui

A react based project that provides a web UI for various tools.

roboto_api

Python FastAPI project that supports multiple API interfaces.

air

A gps location specific weather, and air quality tracking system. Air uses Roboto as a web UI, but can also run as a wh00t chat bot, or simply a data collecting service which dumps weather data into an SQLite DB. The purpose of Air to help people which are sensitive to weather changes, (due to allergies, migraines, ear aches, joint pain, etc) track weather and air quality patterns to see how the weather impacts their issues.

air_core

Core lib used to standardized certain concerns on multiple air projects.

air_collect

Weather data collection service which saves data to CSVs and SQLite DB.

air_bot

Wh00t chat bot that uses air to look up current weather information.

lexicon

A personalized english dictionary made up of the words you curate. Lexicon uses Roboto as a web UI, but can also run as a wh00t chat bot, or as a terminal app. Lexicon helps you learn the words do not know by looking up the definitions for the unknown words, and saving them in an SQLite DB for later study. A history of the words you have looked up and a "word of the day" feature are implemented.
